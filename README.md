# مشروع شركة الخطيب للمقاولات (Al-Khateeb Construction)

هذا المشروع هو عبارة عن نظام متكامل (Full-stack) لشركة الخطيب للمقاولات، تم بناؤه باستخدام تقنيات حديثة لضمان الأداء العالي والتجربة المميزة للمستخدم.

## هيكلية المشروع

المشروع عبارة عن **Monorepo** يدار بواسطة `pnpm` ويحتوي على الأقسام التالية:

*   **`artifacts/alkhatib-contracting`**: الواجهة الأمامية الرئيسية (Vite + React + Tailwind CSS).
*   **`artifacts/api-server`**: الخادم البرمجي (Express.js) مع دعم `Drizzle ORM`.
*   **`artifacts/mockup-sandbox`**: بيئة تجريبية للمكونات والتصاميم.
*   **`lib/`**: مكتبات مشتركة وأدوات مساعدة.

## متطلبات التشغيل

يجب التأكد من تثبيت الأدوات التالية:
1.  [Node.js](https://nodejs.org/) (الإصدار 18 أو أحدث).
2.  [pnpm](https://pnpm.io/) (لإدارة الحزم والمساحات المشتركة).

## تعليمات البدء والتشغيل

### 1. تثبيت الاعتماديات (Dependencies)

من المجلد الرئيسي للمشروع، قم بتشغيل الأمر التالي:
```bash
pnpm install
```

### 2. تشغيل الواجهة الأمامية (Main Frontend)

انتقل إلى مجلد الواجهة الأمامية وقم بتشغيل خادم التطوير:
```bash
cd artifacts/alkhatib-contracting
pnpm run dev
```

### 3. تشغيل الخادم البرمجي (API Server)

في نافذة طرفية (Terminal) منفصلة، انتقل إلى مجلد الخادم وقم بتشغيل الخادم:
```bash
cd artifacts/api-server
pnpm run dev
```

### 4. تشغيل البيئة التجريبية (Mockup Sandbox)

إذا أردت العمل على التصاميم بشكل منفصل:
```bash
cd artifacts/mockup-sandbox
pnpm run dev
```

## تفاصيل مهمة للتطوير

*   **التنسيق (Styling)**: نستخدم Tailwind CSS v4 مع مكونات مبنية على Radix UI.
*   **إدارة الحالة**: نستخدم `@tanstack/react-query` للتعامل مع البيانات القادمة من API.
*   **قاعدة البيانات**: المشروع مهيأ للعمل مع Drizzle ORM للتعامل مع قاعدة البيانات بشكل سهل وآمن.
*   **المكونات**: جميع المكونات الأساسية متوفرة في مجلد `src/components/ui`.

## التعديلات الأخيرة
تمت إزالة كافة الملفات والتبعيات الخاصة بمنصة Replit لضمان استقلالية المشروع وإمكانية تطويره محلياً أو على أي منصة استضافة أخرى (مثل Vercel أو Netlify) بدون أي تتبع أو قيود خارجية.
